# Superlawyer Backend

NestJS 11 + Prisma 7 (PostgreSQL via `@prisma/adapter-pg`) service. Uses `pnpm` and `zod` for validation via `nestjs-zod`.

## Imports

Absolute imports only — no `./` or `../`. Two root aliases configured in `tsconfig.json`:

- `src/*` → project source under `src/`
- `generated/*` → Prisma client under `generated/` (generated by `prisma generate`, output at `generated/prisma/`)

```ts
// Good
import { PrismaService } from 'src/prisma/prisma.service';
import type { User } from 'generated/prisma/client';

// Bad
import { PrismaService } from '../prisma/prisma.service';
import type { User } from '../../generated/prisma/client';
```

`nest build` (tsc) rewrites these aliases to relative requires in `dist/`, so no runtime path registration is needed.

## Layout

```
src/
  app/             root module, health controller
  config/          @nestjs/config registerAs + zod env validation
  crypto/          iron-webcrypto seal/unseal (global module)
  prisma/          PrismaService (extends PrismaClient)
  users/           UsersService — upsert by email
  oauth-accounts/  OAuthAccountsService — stores sealed OAuth tokens
  google-oauth/    Google OAuth flow (auth URL, callback, token refresh)
  google-docs/     Google Docs read API
  main.ts          bootstrap (URI versioning, global ZodValidationPipe)
generated/prisma/  Prisma client output (gitignored-ish, regenerated)
prisma/schema.prisma
```

## Conventions

- Controllers use URI versioning: `@Controller({ path: '...', version: '1' })`.
- DTOs use `nestjs-zod` (`createZodDto`) — validated by the global `ZodValidationPipe` in `main.ts`.
- Config is accessed via `@Inject(config.KEY)` with `ConfigType<typeof config>` — do not read `process.env` directly outside `src/config/env.validation.ts`.
- OAuth tokens are **always** sealed with `CryptoService` before hitting the DB; unseal on read. Never persist plaintext tokens.
- `PrismaService` is provided by `PrismaModule`; `CryptoModule` is global.

## Commands

- `pnpm start:dev` — watch mode
- `pnpm build` — compile to `dist/`
- `pnpm start:prod` — run compiled `dist/src/main`
- `pnpm lint` / `pnpm format`
- `pnpm test` / `pnpm test:e2e`
- `pnpm exec prisma generate` — regenerate client into `generated/prisma/`
- `pnpm exec prisma migrate dev` — apply schema changes locally
